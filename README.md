## GitHub Actions

<!-- Workflow default ditunjukkan sebagai template. Simpan file ini di `.github/workflows/refresh-readme.yml`. -->

README ini sudah mempersiapkan otomatisasi GitHub Actions untuk refresh statis dan memicu pembaruan konten.

### Contoh Workflow

```yaml
name: Refresh README

on:
  schedule:
    - cron: '0 7 * * *' # Jalankan setiap pagi pukul 07:00 UTC
  workflow_dispatch: {}

jobs:
  refresh-readme:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Git
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"

      - name: Touch README to force refresh
        run: |
          echo "README updated at $(date -u '+%Y-%m-%d %H:%M:%S UTC')" > .github/readme-refresh.timestamp
          git add .github/readme-refresh.timestamp
          git commit --quiet -m "chore: refresh README metadata" || echo "No changes to commit"

      - name: Push changes
        uses: stefanzweifel/git-auto-commit-action@v4
        with:
          commit_message: "chore: refresh README"
          branch: ${{ github.ref_name }}
          file_pattern: ".github/readme-refresh.timestamp"
```

> Kustomisasi workflow di atas dengan menambahkan script build, update gambar `assets/`, atau pembaruan otomatis lain yang diperlukan.

### Kustomisasi README Auto Refresh

- Gunakan action untuk menghasilkan gambar SVG lokal.
- Gunakan token GitHub (`GITHUB_TOKEN`) untuk commit otomatis.
- Tambahkan deteksi perubahan konten agar workflow hanya commit jika ada output baru.

## Catatan Customisasi

### Bagian yang sering diubah

1. **Banner hero**: ubah query string di URL `capsule-render`.
2. **Typing animation**: ganti teks pada `readme-typing-svg`.
3. **Badge dan profil**: update top badges dengan `username` dan repo yang benar.
4. **Stack teknologi**: sesuaikan tabel dengan dependencies nyata di repo.
5. **Lingkungan pengembangan**: tambahkan skrip dan variabel environment dari `package.json`.
6. **Proyek unggulan**: update judul dan detail sesuai fitur terbaru.
7. **Workflow GitHub Actions**: letakkan file `.github/workflows/refresh-readme.yml`.

### Tips pemeliharaan

- Periksa badge dan widget luar secara berkala.
- Jika menggunakan API GitHub Stats, pastikan rate limit tidak terlampaui.
- Simpan file SVG di `assets/` agar tampilan statis tetap stabil.
- Gunakan branch `main` atau `master` sesuai cabang default repository.

- `README.md` — dokumentasi utama

- `CONTRIBUTING.md` — panduan kontribusi lebih lengkap
- `CODE_OF_CONDUCT.md` — etika komunitas
